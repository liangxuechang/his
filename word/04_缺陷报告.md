## 二、详细缺陷列表

### BUG-001: 登录接口缺乏参数校验

| 项目 | 内容 |
|------|------|
| **缺陷编号** | BUG-001 |
| **缺陷标题** | 登录接口缺乏参数校验 |
| **严重程度** | 中等 |
| **优先级** | 高 |
| **缺陷类型** | 参数校验缺失 |
| **所属模块** | LoginController |
| **所属接口** | /login/LoginUser |
| **发现日期** | 2026-03-26 |
| **发现人员** | 自动化测试 |
| **缺陷状态** | 新建 |

#### 缺陷描述
登录接口未对userLoginName和password参数进行非空校验，当传入空值时，系统返回模糊的"用户名或密码错误"提示，而非明确的参数校验错误。

#### 复现步骤
1. 发送GET请求到/login/LoginUser
2. 参数userLoginName为空字符串
3. 观察返回结果

#### 实际结果
返回status="Error"，msg="用户名或密码错误"

#### 期望结果
返回明确的参数校验错误提示，如"用户名不能为空"

#### 修复建议
在Controller层增加参数校验：
```java
if (StringUtils.isEmpty(userLoginName)) {
    result.setStatus("Error");
    result.setMsg("用户名不能为空");
    return result;
}
if (StringUtils.isEmpty(password)) {
    result.setStatus("Error");
    result.setMsg("密码不能为空");
    return result;
}
```

### BUG-003: 新增用户未校验登录名重复

| 项目 | 内容 |
|------|------|
| **缺陷编号** | BUG-003 |
| **缺陷标题** | 新增用户未校验登录名重复 |
| **严重程度** | 严重 |
| **优先级** | 高 |
| **缺陷类型** | 业务逻辑缺陷 |
| **所属模块** | UserController |
| **所属接口** | /user/insert |
| **发现日期** | 2026-03-26 |
| **发现人员** | 自动化测试 |
| **缺陷状态** | 新建 |

#### 缺陷描述
新增用户时未检查userLoginname是否已存在，当插入重复登录名时，会导致数据库唯一约束异常。

#### 复现步骤
1. 先新增一个用户，userLoginname=testdup
2. 再次发送新增请求，userLoginname=testdup
3. 观察返回结果

#### 实际结果
数据库抛出唯一约束冲突异常，返回ERROR状态

#### 期望结果
在插入前检查登录名是否存在，返回友好的错误提示"该登录名已存在"

#### 修复建议
```java
@Override
public Integer insert(User user) {
    // 检查登录名是否已存在
    List<User> existUsers = userMapper.getUserByLoginname(user.getUserLoginname());
    if (existUsers != null && !existUsers.isEmpty()) {
        throw new BusinessException("该登录名已存在");
    }
    userMapper.insert(user);
    return user.getUserId();
}
```

### BUG-007: 删除用户未校验用户是否存在

| 项目 | 内容 |
|------|------|
| **缺陷编号** | BUG-007 |
| **缺陷标题** | 删除用户未校验用户是否存在 |
| **严重程度** | 中等 |

| **缺陷类型** | 业务逻辑缺陷 |
| **所属模块** | UserController |
| **所属接口** | /user/delete |
| **发现日期** | 2026-03-26 |
| **发现人员** | 自动化测试 |
| **缺陷状态** | 新建 |

#### 缺陷描述
删除用户接口未先检查用户是否存在，当传入不存在的ID时，仍然返回"删除用户成功"，但实际上未删除任何数据。

#### 复现步骤
1. 发送GET请求到/user/delete
2. 参数id=999999（不存在的ID）
3. 观察返回结果

#### 实际结果
返回status="OK"，msg="删除用户成功！"

#### 期望结果
返回"用户不存在"的提示信息

#### 修复建议
```java
@RequestMapping("/delete")
public ResultDTO<User> delete(Integer id) {
    ResultDTO resultDTO = new ResultDTO();
    try {
        // 先检查用户是否存在
        User user = userService.getUserById(id);
        if (user == null) {
            resultDTO.setStatus("ERROR");
            resultDTO.setMsg("用户不存在！");
            return resultDTO;
        }
        userService.deleteById(id);
        resultDTO.setStatus("OK");
        resultDTO.setMsg("删除用户成功！");
    } catch (Exception e) {
        e.printStackTrace();
        resultDTO.setStatus("ERROR");
        resultDTO.setMsg("删除用户失败！");
    }
    return resultDTO;
}
```

### BUG-011: 用户接口缺乏登录权限校验

| 项目 | 内容 |
|------|------|
| **缺陷编号** | BUG-011 |
| **缺陷标题** | 用户接口缺乏登录权限校验 |
| **严重程度** | 严重 |
| **优先级** | 高 |
| **缺陷类型** | 权限控制缺失 |
| **所属模块** | UserController |
| **所属接口** | /user/* |
| **发现日期** | 2026-03-26 |
| **发现人员** | 自动化测试 |
| **缺陷状态** | 新建 |

#### 缺陷描述
所有用户管理接口都未进行登录状态校验，未登录用户可以直接调用接口进行操作。

#### 复现步骤
1. 清除登录状态（清除Cookie/Session）
2. 直接发送请求到/user/insert或/user/delete
3. 观察返回结果

#### 实际结果
接口正常响应，未进行权限校验

#### 期望结果
未登录用户应返回401未授权错误或重定向到登录页面

#### 修复建议
1. 增加登录拦截器：
```java
@Component
public class LoginInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) {
        // 校验登录状态
        Object user = request.getSession().getAttribute("user");
        if (user == null) {
            response.setStatus(HttpStatus.UNAUTHORIZED.value());
            return false;
        }
        return true;
    }
}
```

2. 注册拦截器：
```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Autowired
    private LoginInterceptor loginInterceptor;
    
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(loginInterceptor)
                .addPathPatterns("/user/**")
                .excludePathPatterns("/login/**");
    }
}
```
