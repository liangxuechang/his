# 接口文档

## 1. 登录接口

### 1.1 用户登录

**接口地址**：`/hoso/login/LoginUser`

**请求方法**：GET/POST

**请求参数**：

| 参数名 | 类型 | 必须 | 说明 |
|--------|------|------|------|
| userLoginName | String | 是 | 用户登录名 |
| password | String | 是 | 密码 |

**响应示例**：
```json
{
    "status": "OK",
    "msg": "请求成功",
    "data": {
        "user": {
            "userId": 1,
            "userLoginname": "admin",
            "userPassword": "123456",
            "roleId": 1,
            "userName": "管理员",
            "departmentId": 1,
            "userTitleId": 1,
            "userGender": "男",
            "userStatus": "正常",
            "userSchedulingLimitcount": 10
        },
        "department": {
            "departmentId": 1,
            "departmentName": "内科"
        },
        "role": {
            "roleId": 1,
            "roleName": "系统管理员"
        }
    }
}
```

**错误响应**：
```json
{
    "status": "Error",
    "msg": "用户名或密码错误",
    "data": null
}
```

## 2. 用户管理接口

### 2.1 新增用户

**接口地址**：`/hoso/user/insert`

**请求方法**：POST

**Content-Type**：application/json

**请求参数**：

| 参数名 | 类型 | 必须 | 说明 |
|--------|------|------|------|
| userLoginname | String | 是 | 用户登录名 |
| userPassword | String | 是 | 密码 |
| roleId | Integer | 是 | 角色ID |
| userName | String | 是 | 用户姓名 |
| departmentId | Integer | 否 | 科室ID |
| userTitleId | Integer | 否 | 职称ID |
| userGender | String | 否 | 性别（男/女） |
| userStatus | String | 否 | 状态 |
| userSchedulingLimitcount | Integer | 否 | 排班限制数 |

**请求示例**：
```json
{
    "userLoginname": "test001",
    "userPassword": "123456",
    "roleId": 2,
    "userName": "测试用户",
    "departmentId": 5,
    "userTitleId": 3,
    "userGender": "男",
    "userStatus": "正常",
    "userSchedulingLimitcount": 20
}
```

**响应示例**：
```json
{
    "status": "OK",
    "msg": "插入用户成功！",
    "data": {
        "userId": 100,
        "userLoginname": "test001",
        "userPassword": "123456",
        "roleId": 2,
        "userName": "测试用户",
        "departmentId": 5,
        "userTitleId": 3,
        "userGender": "男",
        "userStatus": "正常",
        "userSchedulingLimitcount": 20
    }
}
```

---

### 2.2 删除用户

**接口地址**：`/hoso/user/delete`

**请求方法**：GET/POST

**请求参数**：

| 参数名 | 类型 | 必须 | 说明 |
|--------|------|------|------|
| id | Integer | 是 | 用户ID |

**响应示例**：
```json
{
    "status": "OK",
    "msg": "删除用户成功！",
    "data": null
}
```

---

### 2.3 更新用户

**接口地址**：`/hoso/user/update`

**请求方法**：POST

**Content-Type**：application/json

**说明**：更新时必须传入userId

---

### 2.4 查询用户（按ID）

**接口地址**：`/hoso/user/getUserById`

**请求参数**：id（用户ID）

---

### 2.5 查询所有用户

**接口地址**：`/hoso/user/getAllUser`

---

### 2.6 查询所有用户（带角色信息）

**接口地址**：`/hoso/user/getAllUserWithRole`

## 3. 数据模型

### User实体类

| 字段名 | 类型 | 说明 |
|--------|------|------|
| userId | Integer | 用户ID（主键） |
| userLoginname | String | 登录名 |
| userPassword | String | 密码 |
| roleId | Integer | 角色ID |
| userName | String | 姓名 |
| departmentId | Integer | 科室ID |
| userTitleId | Integer | 职称ID |
| userGender | String | 性别 |
| userStatus | String | 状态 |
| userSchedulingLimitcount | Integer | 排班限制数 |

---

## 4. 通用响应结构

所有接口响应均采用统一格式：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| status | String | 状态（OK/ERROR/Error） |
| msg | String | 提示信息 |
| data | T | 返回数据（泛型） |
