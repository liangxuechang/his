# HIS医院信息管理系统 - 接口文档

## 文档信息
- **项目名称**: HIS医院信息管理系统
- **版本**: V1.0
- **编写日期**: 2026年3月26日
- **编写人**: 测试团队

---

## 1. 系统概述

### 1.1 系统简介
HIS医院信息管理系统是一个基于Spring Boot的医院信息管理系统，提供用户管理、挂号管理、门诊医生工作站、医技工作站、药房管理、收费管理等功能模块。

### 1.2 技术架构
- **后端框架**: Spring Boot 2.1.5
- **持久层框架**: MyBatis
- **数据库**: MySQL
- **缓存**: Redis
- **构建工具**: Maven

### 1.3 接口基础信息
- **服务地址**: http://localhost:8080/hoso
- **接口协议**: HTTP/HTTPS
- **数据格式**: JSON
- **编码格式**: UTF-8

## 2. 登录接口

### 2.1 用户登录

**接口描述**: 用户登录系统，验证用户名和密码

**请求URL**: `/login/LoginUser`

**请求方式**: POST/GET

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| userLoginName | String | 是 | 用户登录名 |
| password | String | 是 | 用户密码 |

**请求示例**:
```
POST /hoso/login/LoginUser?userLoginName=admin&password=admin123
```

**响应参数**:

| 参数名 | 类型 | 说明 |
|--------|------|------|
| status | String | 状态码，OK表示成功，Error表示失败 |
| msg | String | 响应消息 |
| data | Object | 返回数据（LoginResult对象） |
| data.user | Object | 用户信息 |
| data.department | Object | 部门信息 |
| data.role | Object | 角色信息 |

**响应示例**:
```json
{
    "status": "OK",
    "msg": "请求成功",
    "data": {
        "user": {
            "userId": 1,
            "userLoginname": "admin",
            "userPassword": "admin123",
            "roleId": 1,
            "userName": "系统管理员",
            "departmentId": null,
            "userTitleId": null,
            "userGender": "男",
            "userStatus": "ACTIVE",
            "userSchedulingLimitcount": 0
        },
        "department": null,
        "role": {
            "roleId": 1,
            "functionId": null,
            "roleName": "系统管理员"
        }
    }
}
```

**错误码说明**:

| 错误码 | 说明 |
|--------|------|
| Error | 用户名或密码错误 |

## 3. 用户管理接口

### 3.1 新增用户

**接口描述**: 新增用户信息到系统

**请求URL**: `/user/insert`

**请求方式**: POST

**请求头**:
```
Content-Type: application/json
```

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| userLoginname | String | 是 | 用户登录名，最大长度50 |
| userPassword | String | 是 | 用户密码，最大长度100 |
| roleId | Integer | 是 | 角色ID，关联role表 |
| userName | String | 是 | 用户姓名，最大长度50 |
| departmentId | Integer | 否 | 科室ID，关联department表 |
| userTitleId | Integer | 否 | 用户职称ID，关联constant_items表 |
| userGender | String | 否 | 用户性别：男/女 |
| userStatus | String | 否 | 用户状态：ACTIVE-在职，INACTIVE-离职 |
| userSchedulingLimitcount | Integer | 否 | 排班限制人数，默认50 |

**请求示例**:
```json
{
    "userLoginname": "testuser001",
    "userPassword": "123456",
    "roleId": 3,
    "userName": "测试用户",
    "departmentId": 1,
    "userTitleId": 9,
    "userGender": "男",
    "userStatus": "ACTIVE",
    "userSchedulingLimitcount": 50
}
```

**响应参数**:

| 参数名 | 类型 | 说明 |
|--------|------|------|
| status | String | 状态码，OK表示成功，ERROR表示失败 |
| msg | String | 响应消息 |
| data | Object | 返回新增的用户对象（包含自动生成的userId） |

**响应示例（成功）**:
```json
{
    "status": "OK",
    "msg": "插入用户成功！",
    "data": {
        "userId": 12,
        "userLoginname": "testuser001",
        "userPassword": "123456",
        "roleId": 3,
        "userName": "测试用户",
        "departmentId": 1,
        "userTitleId": 9,
        "userGender": "男",
        "userStatus": "ACTIVE",
        "userSchedulingLimitcount": 50
    }
}
```

**响应示例（失败）**:
```json
{
    "status": "ERROR",
    "msg": "插入用户失败！",
    "data": null
}
```

## 6. 注意事项

1. 所有接口均支持跨域访问（@CrossOrigin）
2. 用户密码在数据库中以明文存储，建议在生产环境中使用加密存储
3. 删除用户操作为物理删除，请谨慎操作
4. 新增用户时，userId由数据库自动生成
5. 接口未做登录状态校验，建议在生产环境中添加登录拦截器
