## 1. 登录接口

### 1.1 用户登录
- **接口URL**: `/login/LoginUser`
- **请求方式**: GET
- **Content-Type**: application/x-www-form-urlencoded

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| userLoginName | String | 是 | 用户登录名 |
| password | String | 是 | 用户密码 |

#### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| status | String | 状态码：OK/Error |
| msg | String | 提示信息 |
| data | Object | 登录成功返回用户信息 |

#### 响应示例

**成功响应：**
```json
{
  "status": "OK",
  "msg": "请求成功",
  "data": {
    "user": {
      "userId": 1,
      "userLoginname": "admin",
      "userName": "管理员",
      "roleId": 1,
      "departmentId": 1
    },
    "department": {
      "departmentId": 1,
      "departmentName": "信息科"
    },
    "role": {
      "roleId": 1,
      "roleName": "系统管理员"
    }
  }
}
```

**失败响应：**
```json
{
  "status": "Error",
  "msg": "用户名或密码错误",
  "data": null
}
```

## 2. 用户管理接口

### 2.1 新增用户
- **接口URL**: `/user/insert`
- **请求方式**: POST
- **Content-Type**: application/json

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| userId | Integer | 否 | 用户ID（自增，可不传） |
| userLoginname | String | 是 | 用户登录名 |
| userPassword | String | 是 | 用户密码 |
| roleId | Integer | 是 | 角色ID |
| userName | String | 是 | 用户姓名 |
| departmentId | Integer | 是 | 科室ID |
| userTitleId | Integer | 否 | 职称ID |
| userGender | String | 否 | 性别（男/女） |
| userStatus | String | 否 | 用户状态 |
| userSchedulingLimitcount | Integer | 否 | 排班限制数量 |

#### 请求示例
```json
{
  "userLoginname": "zhangsan",
  "userPassword": "123456",
  "roleId": 2,
  "userName": "张三",
  "departmentId": 1,
  "userTitleId": 1,
  "userGender": "男",
  "userStatus": "1",
  "userSchedulingLimitcount": 10
}
```

#### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| status | String | 状态码：OK/ERROR |
| msg | String | 提示信息 |
| data | Object | 插入的用户信息（包含生成的userId） |

#### 响应示例

**成功响应：**
```json
{
  "status": "OK",
  "msg": "插入用户成功！",
  "data": {
    "userId": 100,
    "userLoginname": "zhangsan",
    "userPassword": "123456",
    "roleId": 2,
    "userName": "张三",
    "departmentId": 1,
    "userTitleId": 1,
    "userGender": "男",
    "userStatus": "1",
    "userSchedulingLimitcount": 10
  }
}
```

**失败响应：**
```json
{
  "status": "ERROR",
  "msg": "插入用户失败！",
  "data": null
}
```

## 3. 数据模型

### 3.2 ResultDTO 统一响应模型

| 字段名 | 类型 | 说明 |
|--------|------|------|
| status | String | 状态：OK/ERROR |
| msg | String | 消息提示 |
| data | T | 数据对象 |

## 4. 接口说明

### 4.1 通用说明
1. 所有接口支持跨域访问（@CrossOrigin）
2. 统一返回ResultDTO格式
3. 异常统一捕获处理，返回ERROR状态

### 4.2 状态码说明

| 状态码 | 说明 |
|--------|------|
| OK | 操作成功 |
| ERROR | 操作失败/异常 |
| Error | 登录失败（用户名或密码错误） |

### 4.3 测试环境
- **Base URL**: http://localhost:8080
- **测试账号**: admin / 123456
