## 三、测试执行方式

### 3.1 使用Maven执行测试

```bash
# 执行所有测试
mvn test

# 执行指定测试类
mvn test -Dtest=UserControllerTest
mvn test -Dtest=UserControllerPerformanceTest
mvn test -Dtest=UserServiceTest

# 执行指定测试方法
mvn test -Dtest=UserControllerTest#testLoginSuccess

# 生成测试报告
mvn test jacoco:report
```

### 3.2 使用IDE执行测试

**IntelliJ IDEA**:
1. 右键点击测试类 -> Run 'UserControllerTest'
2. 右键点击测试方法 -> Run 'testLoginSuccess()'

**Eclipse**:
1. 右键点击测试类 -> Run As -> JUnit Test
2. 右键点击测试方法 -> Run As -> JUnit Test

### 3.3 使用测试运行器

```bash
# 编译测试代码
mvn test-compile

# 运行测试运行器
mvn exec:java -Dexec.mainClass="edu.neu.hoso.controller.UserControllerTestRunner" -Dexec.classpathScope=test
```

### 5.2 测试数据工具类

```java
@Component
public class TestDataUtil {
    
    @Autowired
    private UserService userService;
    
    /**
     * 创建测试用户
     */
    public User createTestUser() {
        User user = new User();
        user.setUserLoginname("test" + System.currentTimeMillis());
        user.setUserPassword("123456");
        user.setRoleId(2);
        user.setUserName("测试用户");
        user.setDepartmentId(1);
        return user;
    }
    
    /**
     * 批量创建测试用户
     */
    public List<User> createTestUsers(int count) {
        List<User> users = new ArrayList<>();
        for (int i = 0; i < count; i++) {
            users.add(createTestUser());
        }
        return users;
    }
    
    /**
     * 清理测试数据
     */
    public void cleanTestData() {
        // 清理以test开头的用户
        List<User> testUsers = userService.getUserByLoginname("test");
        for (User user : testUsers) {
            userService.deleteById(user.getUserId());
        }
    }
}
```

## 六、测试报告解读

### 6.1 单元测试报告

```
Tests run: 23, Failures: 0, Errors: 0, Skipped: 0
```

- **Tests run**: 运行的测试总数
- **Failures**: 断言失败的测试数
- **Errors**: 抛出异常的测试数
- **Skipped**: 跳过的测试数

### 6.2 性能测试报告

```
========== 登录接口并发性能测试 ==========

--- 并发用户数: 100 ---
总请求数: 28920
成功请求: 28920
失败请求: 0
平均响应时间: 156.32 ms
最小响应时间: 28 ms
最大响应时间: 892 ms
吞吐量(TPS): 96.40
错误率: 0.00%
```

**关键指标说明**:
- **平均响应时间**: 所有请求的平均响应时间，应 < 500ms
- **吞吐量(TPS)**: 每秒处理的事务数，应 > 50
- **错误率**: 失败请求占比，应 < 1%

## 七、常见问题排查

### 7.1 测试失败排查

| 问题现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 所有测试失败 | 数据库连接失败 | 检查数据库配置 |
| 部分测试失败 | 测试数据冲突 | 使用@Transactional注解 |
| 性能测试异常 | 并发数过高 | 降低并发数或增加超时时间 |
| 登录测试失败 | Session未正确设置 | 检查setUp方法 |

### 7.2 性能优化建议

1. **数据库优化**
   - 添加合适的索引
   - 使用连接池（HikariCP）
   - 优化SQL语句

2. **应用优化**
   - 启用Redis缓存
   - 使用异步处理
   - 调整JVM参数

3. **测试优化**
   - 使用内存数据库（H2）进行单元测试
   - 分离单元测试和集成测试
   - 使用Mock对象减少依赖
