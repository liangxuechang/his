## 2. 单元测试

### 2.1 测试框架

- **JUnit 4**: 测试框架
- **Mockito**: Mock框架
- **Spring Boot Test**: Spring测试支持

### 2.2 依赖配置

在 `pom.xml` 中添加以下依赖：

```xml
<dependencies>
    <!-- Spring Boot Test -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-test</artifactId>
        <scope>test</scope>
    </dependency>
    
    <!-- JUnit -->
    <dependency>
        <groupId>junit</groupId>
        <artifactId>junit</artifactId>
        <scope>test</scope>
    </dependency>
    
    <!-- Mockito -->
    <dependency>
        <groupId>org.mockito</groupId>
        <artifactId>mockito-core</artifactId>
        <scope>test</scope>
    </dependency>
</dependencies>
```

### 2.3 运行单元测试

**方式一：Maven命令**

```bash
# 运行所有测试
mvn test

# 运行指定测试类
mvn test -Dtest=UserControllerTest

# 运行指定测试方法
mvn test -Dtest=UserControllerTest#testInsert_Success
```

**方式二：IDE运行**

在IDE中右键点击测试类或测试方法，选择"Run"即可执行。

## 4. 性能测试

### 4.1 测试工具

- **Apache JMeter 5.5**: 性能测试工具

### 4.2 测试脚本说明

**HIS_Performance_Test.jmx** 包含以下测试场景：

| 线程组 | 并发数 | 循环次数 | 测试接口 |
|--------|--------|----------|----------|
| 登录接口测试 | 100 | 10 | /login/LoginUser |
| 新增用户接口测试 | 50 | 10 | /user/insert |
| 删除用户接口测试 | 50 | 10 | /user/delete |

### 4.3 运行性能测试

**方式一：GUI模式**

```bash
jmeter -t HIS_Performance_Test.jmx
```

**方式二：命令行模式**

```bash
jmeter -n -t HIS_Performance_Test.jmx -l results.jtl -e -o report
```

参数说明：
- `-n`: 非GUI模式
- `-t`: 测试计划文件
- `-l`: 结果文件
- `-e`: 生成HTML报告
- `-o`: 报告输出目录

### 4.4 修改测试参数

在JMeter中修改以下变量：

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| BASE_URL | http://localhost:8080/hoso | 服务地址 |

## 6. 测试执行流程

### 6.1 推荐执行顺序

```
1. 单元测试
   ├── UserControllerTest.java
   ├── LoginControllerTest.java
   └── UserServiceImplTest.java

2. 集成测试
   ├── LoginControllerIntegrationTest.java
   └── UserControllerIntegrationTest.java

3. 接口测试 (Postman)
   └── HIS_API_Tests.postman_collection.json

4. 性能测试 (JMeter)
   └── HIS_Performance_Test.jmx
```

### 6.2 CI/CD集成

**Jenkins Pipeline 示例**:

```groovy
pipeline {
    agent any
    
    stages {
        stage('Unit Tests') {
            steps {
                sh 'mvn test'
            }
        }
        
        stage('Integration Tests') {
            steps {
                sh 'mvn verify'
            }
        }
        
        stage('Performance Tests') {
            steps {
                sh 'jmeter -n -t word/test-code/HIS_Performance_Test.jmx -l results.jtl'
            }
        }
    }
    
    post {
        always {
            junit '**/target/surefire-reports/*.xml'
            publishHTML(target: [
                reportDir: 'report',
                reportFiles: 'index.html',
                reportName: 'Performance Report'
            ])
        }
    }
}
```

## 8. 注意事项

1. **数据库准备**: 集成测试需要连接真实数据库，请确保数据库已启动并包含测试数据
2. **端口冲突**: 确保8080端口未被占用
3. **测试数据隔离**: 集成测试使用 `@Transactional` 注解，测试完成后自动回滚
4. **性能测试**: 运行性能测试前请确保系统处于空闲状态
5. **并发控制**: 性能测试并发数不宜过高，避免压垮测试环境

## 9. 附录

### 9.1 常见问题

**Q: 单元测试报错找不到类？**

A: 确保已执行 `mvn clean install` 安装所有依赖。

**Q: 集成测试数据库连接失败？**

A: 检查 `application.yml` 中的数据库配置是否正确。

**Q: JMeter无法启动？**

A: 确保已安装JDK 8+并配置JAVA_HOME环境变量。
